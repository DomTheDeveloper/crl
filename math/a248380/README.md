# OEIS A248380(16): odd-response workbench

## Current verdict

The full Sylver Coinage problem after opening `16` is **not solved yet**.
The candidate winning reply is `26`. This package closes the entire infinite
odd-response side computationally and records the exact remaining even frontier.

## Odd-response reduction

Let `S = {8,13}`, so `{16,26} = 2S`. The numerical semigroup generated by
`S` has Frobenius number `83` and exactly `42` gaps. By the Quiet End Theorem,
for odd `q`, `{16,26,q}` is a quiet ender whenever `q` is already illegal in
`S`. Thus only the odd gaps of `S` need finite checking. Move `1` loses
immediately, leaving exactly 23 exceptional odd responses.

`odd_witnesses.json` records an exact winning reply for each exception. The
final and hardest row was:

```
{16,26,83} --127--> {16,26,83,127}  (computed P)
```

All 23 resulting canonical P-positions occur in the exact P-state cache used by
the solver. `verify_odd_reduction.py` checks the semigroup arithmetic, finite
coverage, legality, and canonicalization.

## Verification boundary

The large outcome memo for the final row contains 19,247,759 states and marks
`{16,26,83,127}` as P. A fully independent recursive replay is still pending;
the first replay implementation exceeded memory and is being replaced by a
compact/disk-backed checker. Therefore this package says **computed**, not yet
**independently certified** or **Lean-verified**.

## Even frontier

There are 42 legal even responses to `{16,26}`. Prior work reported 29 closed,
but their certificate artifacts were not recovered. Thirteen rows remain
unresolved:

```
70, 82, 86, 88, 92, 98, 102, 114, 118, 124, 134, 150, 166
```

See `even_rows.json`. The full theorem `a(16)=2` requires replayable closures for
all 42 even rows in addition to the odd-response result above.
